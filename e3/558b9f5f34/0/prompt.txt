5 Python files in `ch05/04_evaluation/` use `click` for CLI argument parsing, while 3 other files (`03_*.py`) already use `argparse`. This change unifies all scripts to use `argparse` (stdlib), eliminating the `click` dependency for this directory.

## Conversion pattern

Each file follows the same transformation:

1. Replace `import click` with `import argparse`
2. Remove all `@click.command()`, `@click.option()`, `@click.argument()` decorators
3. Add `argparse.ArgumentParser()` setup inside the `main()` / `cli()` function
4. Use `argparse.BooleanOptionalAction` (available in Python 3.9+, project requires 3.10) for `--flag/--no-flag` boolean pairs
5. Use positional `add_argument('input_fn', ...)` for click `@click.argument()` equivalents
6. Replace `click.UsageError(...)` with `parser.error(...)` (in `naive_rag.py`)
7. Drop `click.Path(...)` type â€” just use `str` (argparse doesn't have built-in path validation; existing code already handles paths via `pathlib.Path`)
8. Call `args = parser.parse_args()` then pass `args.*` to the async `_main()` function

### Mapping reference

| click feature | argparse equivalent |
|---|---|
| `@click.option('--flag/--no-flag', default=False)` | `add_argument('--flag', action=BooleanOptionalAction, default=False)` |
| `@click.option('--name', '-n', default=-1, type=int)` | `add_argument('--name', '-n', default=-1, type=int)` |
| `@click.option('--index/--no-index', "do_index")` | `add_argument('--index', dest='do_index', action=BooleanOptionalAction)` |
| `@click.option('--x', required=True)` | `add_argument('--x', required=True)` |
| `@click.argument('input_fn', default="x.json")` | `add_argument('input_fn', nargs='?', default="x.json")` |
| `@click.argument('input_fn', type=click.Path(exists=True))` | `add_argument('input_fn')` (just a string) |
| `click.UsageError("msg")` | `parser.error("msg")` |
